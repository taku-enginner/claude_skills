## 応用パターン

```

```

```
テストを書いて。境界値テストは多めに
```

```

```

```

```

```

```

```

```

```

```

```

```

## トリガーワード一覧

Claudeが各スキルを起動するキーワード:

| スキル | トリガーワード例 |
|--------|------------------|
| 動画確認 | 動画を見て、録画を確認、ビデオをチェック、動画の内容 |
| スクショ比較 | 比較して、差分を見せて、違いを教えて、変更点 |
| エラー検出 | エラーを探して、異常を検出、問題を見つけて、バグを探して |
| 実装 | 実装して、作って、作成して、追加して |
| テスト生成 | テストを書いて、テストを生成、テストケースを作って |
| テスト実行 | テストを実行、テストして、テストを走らせて |
| コード解析 | 構造を教えて、解析して、一覧して、調べて |

## ベストプラクティス

### 1. 具体的に指示する

```
❌ テストして
✅ lib/utils/validator.dart のテストを生成して実行して
```

### 2. ファイルパスを明示する

```
❌ この動画を見て
✅ recordings/test_20260126.mp4 を見て
```

### 3. 期待する結果を伝える

```
❌ バリデーションを作って
✅ メールバリデーションを作って。空文字とフォーマット不正でfalseを返して
```

### 4. 制約を伝える

```
❌ テストを書いて
✅ テストを書いて。モックは使わないで、シンプルなユニットテストで
```

## スラッシュコマンドとの対応

自然言語の代わりにスラッシュコマンドも使えます:

| 自然言語 | スラッシュコマンド |
|----------|-------------------|
| 〇〇を実装して | `/project:implement 〇〇` |
| テストを生成して | `/project:generate-tests <file>` |
| テストを実行して | `/project:run-tests` |
| 動画を確認して | `/project:analyze-video <file>` |
| スクショを比較して | `/project:compare-screens <file1> <file2>` |
| エラーを探して | `/project:find-errors <file>` |
